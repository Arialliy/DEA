## 1. SIED

- mean-anchor `alpha=0.2` 在历史 43-val 仅有 `+0.002135` IoU 弱信号；
- PD/FA 不变，只改变 37 个阈值像素；

- decoder0-only Conv-MAC 增量 `33.03%`，超过 25% 上界；
- 只能作为诊断 probe，不能直接成为最终 DEA。

## 2. MA-PCID

- 用 conditional increment 替代 factual inherited state；
- `alpha=1` 时历史 IoU 约 `0.201576`、PD `0.714286`；
- 说明 interaction/difference state 不能替换完整粗尺度 decoder 语义；
- 强 NO-GO。

## 3. Shared Discrepancy Stencil

- 最佳历史学习态 `0.745749 < 0.747056`；
- 另有 PD 降、FA 恶化；
- 共享线性 stencil 缺乏语义条件能力；
- 禁止通过 adapter/attention/channel mixer 继续堆叠救活。

## 4. CEV

- terminal final-fusion attenuation/gate；
- harmfulness teacher 在解析上退化为删除 gate 的标签×贡献符号；
- 不能承担因果或主创新 claim；
- 只保留为 final-fusion control。

## 5. SMEI / component oracle

- 历史 oracle 显示部分 baseline FP 有尺度子集修复空间；
- 对 representation-level FN 没有恢复能力；
- 需要 component pipeline + 16-way router；
- 只能作为能力上界，不能当可部署主模型。

## 6. PredictiveCorrection / DEA v0

- 删除 MSHNet 强 hierarchical decoder，换成 tied-adjoint/Landweber 更新；
- 历史 400 epoch 被 MSHNet 严格支配；
- 禁止再删除 factual decoder 或靠 state width/step size 调参救活。

## 7. DEAIntegrated / CSIR / PSIR

- tri-state routing action 不可识别；
- 无 action supervision 时 target occupancy 为 0；
- 加 supervision 后 collapse 为 100% keep；
- 禁止复活四级 router/terminal closure。

## 8. FullDEA 系列

- terminal target/clutter/counterfactual head 插入点错误；
- v2 主要表现为 FA 降但 PD/IoU 同时下降；
- v3 在 NUAA 仅微增、NUDT 退化且 latency 约 `1.44×`；
- v4/v5/v6 无统一、可审计 paired 性能；
- 禁止继续 topology bridge、component relation selector 和 operation routing 堆叠。

# 八、新 DEA 的硬设计禁区

1. factual MSHNet decoder path 必须始终显式保留；
2. 不得用 difference/interaction/persistent state 替换 inherited decoder state；
3. 不做 terminal gate、router、component pipeline 或 correction head；
4. 不做多 adapter、多 attention、多 residual 的模块堆叠；
5. 不用可学习全局 `alpha`、ReZero 或普通 residual 包装创新；
6. 不把 Möbius/Shapley/ANOVA 分解本身当结构创新；
7. 不因 FA 下降而接受 PD/TP 损失；
8. 不把 oracle、shape test、identity test 当性能证据；
9. 不复活已被完整实验否定的 DEA v0、MA-PCID、stencil、tri-state route 或 topology bridge；
10. 新机制必须直接对应 clean baseline 中跨数据集复现的根因。

允许继承：

- MSHNet factual encoder/decoder；
- mean-anchor interaction 作为无 GT 诊断变量；
- CEV、ordinary residual、attention、final-fusion 作为 controls；
- component oracle 作为能力上界；
- P0 identity、split hash、paired seed 和 checkpoint metadata 工程合同。

# 九、预定义的 decoder-interaction 机制审计

设 finest decoder transition 的 current skip 为 `e`，inherited decoder state 为 `u`，原 decoder 为 `D(e,u)`。

Mean anchor：

\[
\bar e=\operatorname{sg}[\operatorname{Mean}_{xy}(e)],\qquad
\bar u=\operatorname{sg}[\operatorname{Mean}_{xy}(u)].
\]

四个 coalition：

\[
q^{11}=D(e,u),\quad
q^{10}=D(e,\bar u),\quad
q^{01}=D(\bar e,u),\quad
q^{00}=D(\bar e,\bar u).
\]

当前独立项与 interaction：

\[
p=q^{10}-q^{00},\qquad
j=q^{11}-q^{10}-q^{01}+q^{00}.
\]

沿原 `output_0 + final` 线性路径投影后：

\[
p_z=z^{10}-z^{00},\qquad
j_z=z^{11}-z^{10}-z^{01}+z^{00}.
\]

像素级 interaction ratio：

\[
r(x)=\frac{\operatorname{RMS}_{c}(j(x))}
{\operatorname{RMS}_{c}(p(x))+\epsilon}.
\]

无 GT conflict score：

\[
C(x)=\log(1+r(x))\,
\mathbf 1[p_z(x)j_z(x)<0].
\]

无 GT conflict mask：

\[
M(x)=
\mathbf 1[
r(x)\ge1
\land p_z(x)j_z(x)<0
\land |p_z(x)|+|j_z(x)|>\epsilon
].
\]

预注册的图像级 benefit predictor：

\[
I_{\mathrm{mean-anchor}}(X)
=\frac{1}{HW}\sum_x C(x).
\]

该指标不读取 GT。候选训练完成后，用它预测：

\[
\mathbf 1[\Delta\operatorname{IoU}_{image}>0].
\]

在独立 seed/fold/dataset 上报告 AUROC、Spearman 和 clustered bootstrap；不能把同一图像的三个 seed 当成三个独立样本。

## 每图必须导出的证据

- intersection / union；
- GT/pred positive pixels；
- pixel FP/FN；
- target、matched target、FN target 数量；
- prediction、matched prediction、FP component 数量；
- unmatched FP component area；
- TP/FN/FP component IDs、centroid、bbox、match distance；
- recoverable FN IDs；
- mean-anchor index；
- interaction ratio mean/p95；
- conflict pixels/fraction；
- `z11/z10/z01/z00`、ratio、conflict score/mask 数组。

全局指标必须使用 ratio-of-sums：

\[
\mathrm{IoU}=\frac{\sum_i intersection_i}{\sum_i union_i},
\]

而不是平均 per-image IoU。

# 十、DEA 设计交付合同

Baseline 与机制审计完成后，DEA 设计稿必须包含：

## 1. Problem

- 三个数据集共同出现的具体 MSHNet failure；
- 失败主要对应 FP、FN、边界偏差、组件碎裂或 decoder conflict 中的哪一种；
- 该失败的频率、效应量与 seed 稳定性。

## 2. Gap

- MSHNet 原 fusion/recurrence 为什么造成该失败；
- ordinary residual 为什么只做 calibration；
- attention 为什么不能表达所需的结构约束；
- final-fusion gate 为什么作用位置太晚；
- 历史 correction state 为什么破坏 factual path；
- component router 为什么不是端到端 representation-level 解法。

## 3. Core insight

- 用一个明确机制解释如何在不破坏 factual decoder state 的前提下修复根因；
- insight 必须能形成可证伪预测，而不是抽象命名。

## 4. Method

- 只允许一个主机制；
- 给出公式、数据流和 state transition；
- 关闭时 bitwise identity；
- 不能依赖全局 learnable alpha；
- 不增加第二套 encoder/decoder；
- 不使用 router/gate/head 链条救性能。

## 5. Complexity

- 参数增量目标不超过 10%；
- MACs/FLOPs 增量不超过 25%；
- latency 增量不超过 30%；
- 在写完整训练代码前先给出复杂度上界。

## 6. Controls

至少包含参数量匹配的：

1. ordinary residual；
2. attention；
3. final-fusion control；
4. identity/alpha-off baseline；
5. 必要时 fixed SIED probe。

# 十一、性能与机制门

## P0：Identity

关闭机制时与 MSHNet bitwise identity。

## P1：三数据集开发 holdout

每个数据集 paired：

\[
\Delta IoU>0,\qquad
\Delta PD\ge0,\qquad
\Delta FA/M\le0.
\]

## P2：稳定性

- OOF / unseen fold；
- paired image bootstrap；
- 多 seed；
- 方向不由单图或单 fold 主导；
- mechanism index 具有 predictive validity。

## P3a：最低存活

官方 test、多 seed：

- mean `Delta IoU >= +0.005`；
- 每个 seed `Delta PD >= 0`；
- mean `FA_DEA / FA_MSHNet <= 0.90`；
- 至少 2/3 seeds Pareto 支配 baseline。

## P3b：顶会级主结果

- mean `Delta IoU >= +0.008`；
- 目标区间 `+0.008～+0.015`；
- mean `Delta PD > 0`；
- mean `FA_DEA / FA_MSHNet <= 0.85`；
- FA/M 目标下降 `15%～30%`；
- paired bootstrap 的 IoU 95% CI 下界 `>0`。

## P4：跨数据集

- NUAA-SIRST、NUDT-SIRST、IRSTD-1K 全部运行；
- 三个数据集 mean `Delta IoU` 全为正；
- 至少两个数据集 `Delta IoU >= +0.008`；
- PD 不降；
- 跨数据集平均 FA 下降至少 15%。

## P5：效率

- 参数增量 `<=10%`；
- MACs/FLOPs 增量 `<=25%`；
- latency 增量 `<=30%`。

## 机制归因合同

即使 P3b 通过，也必须额外证明：

1. TP preservation；
2. baseline FP removal；
3. recoverable FN recovery；
4. conflict localization；
5. mean-anchor predictive validity；
6. structural specificity against residual/attention/final-fusion controls。

- train/test ID 只做交集/hash 审计；
- 训练循环只调用 `train(epoch)`；
- 不调用 validation/test；
- 固定训练到最后 epoch；
- 保存 `checkpoint_last.pkl`；
- 不写 best-validation 指标；
- 最终 test 在独立进程、独立 manifest 中统一解封。

Final checkpoint 应记录：

```text

train/test split hash and counts

model_type

completed=true
```

最终 MSHNet 与 DEA 必须在三个数据集、相同 seeds、相同 fixed-last 规则下全部重训。

# 十三、停止条件

以下任一成立，应终止当前问题或模型：

- 增益只在一个 checkpoint、一个 seed 或一个数据集出现；
- improvement 被一张图、一个 fold 或 design fold 主导；
- PD 减少任何目标；
- FA 明显恶化；
- 需要重新搜索 alpha、anchor、stage 或 threshold 才为正；
- 需要解冻整个 MSHNet 才能涨点；
- ordinary residual、attention 或 final-fusion control 复现全部收益；
- mean-anchor 指标不能预测受益；
- 改善不集中在预定义 conflict 区域；
- 只能消除 FP，却持续损害 TP/PD；
- 复杂度超过 P5；
- 需要增加第二、第三个模块救性能。

若 interaction 假设失败，只能依据 clean baseline 的另一项独立、跨数据集证据重新定义问题；若不存在，则 DEA 设计 NO-GO。
